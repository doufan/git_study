思考：手动与自动拉取
现在，既然你已将更改推送到 GitHub 并从 GitHub 拉取了更改，现在请将以下问题和你对它的想法添加到你的思考文件中：

为何你希望始终手动地拉取更改，而不是让 Git 自动与你的远程版本库保持同步？
答：
自动拉取的并不一定是我希望合并到我的本地版本中的，所以，还是使用手动拉取，随时拉取自己想要拉取的内容。


14.思考：Fork、克隆和分支
现在，你已了解如何通过将 GitHub 上的版本库Fork来创建其副本，现在请将以下问题和你对它的想法添加到你的反思文件中：

说明Fork、克隆和分支之间的不同。何时使用某种功能而不是另一种？
答：
fork 是用在GitHub上将别人的远程代码库复制一份到我的GitHub上。
clone 是将GitHub上的一个远程代码库复制一份到我的本地。
分支 是一个代码库内的多个分支。
